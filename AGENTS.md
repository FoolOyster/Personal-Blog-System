# 个人博客系统开发指令

## 项目概述

拥有完整前后端的个人博客系统

## 开发规范

- 错误要有友好的提示
- 组件要可复用
- 代码要有注释-中文注释
- 请使用中文来写文档

## 代码风格

- 使用 ESLint 和 Prettier
- 组件名使用 PascalCase
- 函数名使用 camelCase
- 常量使用 UPPER_SNAKE_CASE

## 设计要求

- 保持明亮界面，极光渐变主题
- 各种交互有流畅的动画效果
- 确保移动端适配

## 测试要求

- 每个功能完成后手动测试
- 确保数据正确存储和读取
- 测试各种边界情况

## 注意事项

- 要处理 API 调用失败的情况
- 加载时要有明确的提示

## 代码提交

- 每次完成有意义的代码或文件修改后，使用 git commit 指令提交修改
- 自主修改 .gitignore 文件，项目中依赖文件或日志文件或测试功能等代码或环境变量文件无需提交
- 提交命名请遵照：feat/chore/docs/fix/others: 中文描述
